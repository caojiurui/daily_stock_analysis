# 个人投资助手能力缺口与对标分析

本文档用于收口路线图 `5.11` / `Phase 7`，基于 2026-06-21 对当前仓库、官方产品资料和成熟 research workflow 的对照，回答三件事：

- 当前系统已经擅长什么
- 与成熟产品相比还缺什么
- 哪些值得补，哪些不值得复制

## 1. 对标范围

本轮对标只选择和当前仓库目标最相关的两类对象：

- 面向投资研究 / 机会发现的产品形态
- 面向研究、回测、自动化的 workflow 形态

不把“通用券商 App”“完整量化交易云平台”“社交投资社区”当成必须复制的目标。

## 2. 当前仓库已具备的优势

围绕“个人投资助手”目标，仓库当前已经形成了一个很明确的最小闭环：

- 机会摘要：`/api/v1/opportunities/overview`
- 候选输出：个股 / ETF 候选、`cycle_view`、`portfolio_advice`
- 监控闭环：机会类 alert evaluator、worker、通知与节流
- 联动工作台：`/screening`、`/portfolio`、`/alerts`
- 题材证据：AlphaSift、新闻搜索、THS 外部信号、复盘历史

这意味着当前系统已经不是“单次个股分析脚本”，而是一个轻量、可旁路、可降级的机会工作台。

## 3. 成熟产品对标

### 3.1 值得学习的产品特征

成熟产品的共性不是“指标更多”，而是“研究动作被组织得更顺手”。

建议重点吸收以下模式：

- 自选 / 组合 / 告警 / 新闻放在同一工作区，而不是分散在多个入口
- 允许从 watchlist 直接观察技术、新闻和阈值变化
- 允许在一个主题下快速切 ETF、成分股、相关新闻与说明
- 输出不仅有结果，还要有可复查的证据与路线

### 3.2 不值得照搬的产品特征

以下能力对当前仓库的个人投资助手定位并不划算：

- 重社区、重社交、重内容分发
- 大而全的券商交易入口
- 面向机构协作的大型权限与审批系统
- 通用量化云平台式的完整研究托管与执行基础设施

## 4. research workflow 对标

成熟 research workflow 的共性是把研究过程拆成清晰层次：

- 数据接入层
- 统一分析 / 归一化层
- 研究与解释层
- 回测 / 复盘层
- 监控 / 告警层
- 结果消费层（UI / 报告 / 代理）

当前仓库已经覆盖了其中大部分层次，但还缺少两类“可重复研究资产”：

- 更体系化的 skill / agent 层
- 更清晰的能力缺口与优先级文档

## 5. 当前主要缺口

### 5.1 产品层缺口

- 缺统一的“主题研究页”或“研究工作区”
- 缺更强的 watchlist 模板化能力
- 缺更细的自定义筛选器与筛选结果持久化
- 缺更清晰的外部证据来源标识与数据质量解释
- 缺面向新手的一致化解释模板

### 5.2 research workflow 缺口

- 缺正式的 skill 体系来复用常见研究动作
- 缺“主题 -> ETF -> 个股 -> 告警 -> 复盘”的标准作业说明
- 缺更体系化的实验 / 回测入口
- 缺更明确的数据血缘、来源解释与质量分层

### 5.3 工程与治理缺口

- Web/后端联动测试仍有局部缺口
- 前端 `vitest` 环境仍存在 ESM 兼容阻塞
- 外部信息源虽已扩展，但来源优先级与展示解释仍可继续标准化

## 6. 哪些能力值得复制

优先级最高：

- 统一工作区视角：把机会、自选、告警、组合判断组织得更顺
- watchlist 驱动的模板化研究动作
- 证据透明化：来源、时效、降级、warning 解释
- 新手友好的行动建议格式
- 更稳定的 skill / agent 复用层

中优先级：

- 主题研究页或研究 tab
- 更细筛选器与保存后的复用扫描模板
- 面向 ETF 与主题轮动的对比视图

低优先级：

- 更复杂的图表工作台
- 通用笔记本 / 云研究环境
- 完整量化回测平台能力

## 7. 哪些不值得复制

- 券商交易执行闭环
- 社交社区和观点广场
- 广义机构级权限系统
- 高成本的多租户量化托管平台

这些方向会显著拉高复杂度，但并不直接提升当前“个人投资助手”主目标。

## 8. 建议收口结论

当前仓库长期上最合适的路线不是“做一个更大的股票网站”，而是继续收敛成：

- 一个以机会发现为中心的个人研究工作台
- 一个以 watchlist / portfolio / alerts 为主线的决策辅助系统
- 一个可复用、可解释、可降级的 agent + skill 层

## 9. 建议下一步

推荐继续顺序：

1. 先把 `5.10` skill 体系真正用起来
2. 再补一页轻量“主题研究工作区”或等价联动入口
3. 再补数据质量 / 来源解释的一致化展示
4. 最后才考虑更重的研究与回测基础设施
